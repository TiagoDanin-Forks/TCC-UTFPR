# Etapa 1: Coletar os seguintes dados:
* Número de novatos mensais (via Repositório)
* Número de contribuições mensais (via Repositório)
* Número de pull-requests mensais (via GitHub API)
* Número de estrelas mensais (via GitHub API)
* Número de cópias mensais (via GitHub API)

Neste etapa, os dados coletados devem ser mensalmente distribuídos, ou seja, será necessário coletar a data de cada 
ocorrência para cada categoria coletada. O objetivo ao fim desta etapa é realizar uma comparação mensal entre as categorias, por exemplo, comparar o número de novatos em um mês em relação ao número de estrelas, cópias, etc.

## Relatório desta etapa:
Todos os dados foram coletados com sucesso. Para número de contribuições e novatos, que condizem com informações dispostas 
no repositório, foi utilizado uma cópia do próprio repositório e os comandos do "git log" para extrair os dados. Para as demais categorias coletadas, estas relacionadas a informações dispostas no GitHub, foi utilizada a GitHub API.

Coletar novos contribuidores mensais foi um desafio. A ideia era coletar todas as contribuições de um repositório e verificar, para cada contribuidor, a data em que este realizou sua primeira contribuição. Entretanto, as contribuições, quando coletadas via API, não estavam completas para praticamente nenhum dos projetos selecionados. Isso porque, a API só apresenta aos usuários as contribuições que foram realizadas dentro do GitHub. Como foram analisados projetos populares, e em sua grande parte antigos, nem todos começaram nesta plataforma de codificação. Ou seja, em certos casos, o número de contribuições disponível na API não condizia com o númeo real de contribuições apresentado no repositório. O repositório do Linux, por exemplo, só possuia 23 mil contribuições cadastradas via GitHub API, de mais de 600 mil presentes no repositório. Visto este problema, utilizamos o repositório como meio para extrair a data em que cada contribuidor realizou sua primeira contribuição. Esta decisão demandou mais tempo de execução e coleta em relação a uma possível coleta via API, mas funcionou muito bem. Os novatos no repositório do Linux, citado anteriormente, demoraram aproximadamente 24 horas para serem coletados, sendo considerado o maior tempo de coleta entre todos os repositórios.

As contribuições mensais foram facilmente coletadas, visto que elas já haviam sido utilizadas para encontrar os novatos. Só foi necessário criar um arquivo separado disponibilizando elas para uma análise e geração dos gráficos. Os pull requests foram coletados utilizando o Octopus, um coletor de dados que eu desenvolvi anteriormente para a GitHub API. Arquivos JSON foram salvos para cada um dos repositórios contendo todos os pull requests realizados respectivamente em cada um deles. O número de cópias seguiu o mesmo princípio dos pull requests. O de estrelas teve apenas uma alteração: Para coletar a data em que uma estrela foi atribuída a um repositório, foi necessário passar via cabeçalho HTTP a seguinte informação "Accept: application/vnd.github.v3.star+json". Esta informação rendeu algumas pequenas modificações no Octopus, mas nada demais em relação ao código original.

Pode-se dizer que esta etapa foi concluída com sucesso em um curto período de tempo, aproximadamente duas semana. Houveram algumas pequenas barreiras na implementação dos códigos e no armazenamento das informações. O dataset desta etapa ao final de toda a coleta, contando arquivos JSON e cópias dos repositórios, totalizou +15GB em dados.

Ameaças a validade:
    Em pouquíssimos casos o script, via git, retornou um valor vazio para a data dos novatos. Em apenas um caso no repositório do Go, a data de entrada do contribuidor foi coletada como sendo 1976. Análises foram realizadas para avaliar a veracidade dos dados coletados, e para todos os demais casos, as informações condiziam com as presentes nos repositórios.
